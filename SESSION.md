## 2026-05-12 (поздно) — R15 откачено + доработан /audit-connections

**Что произошло:**
Применил 15 двусторонних connections из аудита (Тип А + Тип Б) в ветке `feature/r15-apply-audit`, прошёл `/check` зелёным. На preview оказалось: визуально **ничего не появилось** — нет точек у второго человека (Сталин 1933, Мамонтов 1898, Моне 1905...), нет парных попапов, нет connection-линий. Связь визуально работает **только если у обоих есть `event` на тот же год**, а не просто mutual `connections[]`. Это пропущенная семантика в самом скилле.

**Действие:**
Ветка `feature/r15-apply-audit` удалена локально и на remote, правки people.json не попали в main. Сохранён только `chore(gitignore): exclude .scratch/` (`b757490`) — он независим. `/audit-connections` доработан: теперь делит находки на **G1** (event у обоих → можно сразу применять, визуальный эффект сразу), **G2** (event только у А → нужно сначала создать event у второго или оставить без визуала), **G3** (связь есть на другой год → решать индивидуально). В шапке скилла теперь явное предупреждение о требовании event у обоих.

**Lesson learned:**
Применение «двусторонней связи» в `connections[]` без `event` у второго человека — это нулевой визуальный эффект. R15 в реальности — не только про связи, но и про **парные events**. Это серьёзно меняет объём работы (вместо «добавить connection» — «решить, создавать ли event у Б, какое именно, потом добавить connection»).

**Что отложено:**
Само применение R15 — следующая сессия с свежей головой. Прогон обновлённого `/audit-connections` даст список G1/G2/G3, и можно действовать по группам.
