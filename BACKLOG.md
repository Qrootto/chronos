## Дизайн-долг

### Попап
- **Особый попап для связанных событий (попап-пара)** — когда событие связано с другим человеком (через `connections[]` обоих), при клике на эту точку у любого из двоих открывается **особый попап про их совместное событие**.
  - Пример: «А женился на Б» в 1990 году. У А в `events[]` это событие записано как «женился на Б», у Б — «женился на А», в `connections[]` обоих стоит ссылка друг на друга с year=1990. При клике на эту точку у А или у Б открывается попап про обоих, описывающий их совместное событие (а не два отдельных).
  - Дизайн пока не нарисован. До готовности — показываем стандартный попап.
- **Расширить `events[i]`** — добавить опциональное поле `sections: [{title, body}]` в схему данных для подзаголовков и подробного текста (как в Figma 8097). Пока попап рендерит только `description` как lead.
- **Заполнить тексты** — по 200+ событиям описания сейчас 1-2 предложения. Расширить хотя бы у популярных людей.
- **Рабочие фото для всех 43 человек** — текущие URL'ы в `data/people.json` часто 404. Нужно заменить на стабильные ссылки с Wikimedia Commons / Wikipedia REST API thumbnail (все Public Domain / CC). Сделать одноразовый скрипт обновления.

## Беклог

### Hover на event-dot — возраст
- При наведении на событие в жизни человека отображать, **сколько ему было лет** в этот момент. Подробности уточнит пользователь позже.

### Vertical line при hover года
- При hover на год (любым способом — мышью по timeflow или прямо по метке) над ним рисуется вертикальная линия, проходящая через всю высоту Timeflow.
- Там, где линия пересекает чью-то life-line, появляется метка с возрастом этого человека в данный год.
- Цель: пользователь сразу видит «вот этим людям тогда было столько-то лет».

- **Я.Метрика**: счётчик + cookie banner + страница «Политика конфиденциальности». Нужен Counter ID от пользователя.
