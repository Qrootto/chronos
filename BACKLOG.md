## К релизу

- [ ] **R1** Факт-чек текстов (приоритет №1)
  Все 181 секций (`events.sections`) и `events.description` в `public/data/people.json` — черновики Claude. Сверить с Wikipedia / Britannica перед production. Сэмпл из 15 секций показал ~27% брака. Конкретный замеченный кейс — **Ленин 1905** (по нашим данным вернулся в Россию; проверить, действительно ли возвращался в этом году) — и качественный фактчек остальных событий.

- [ ] **R2** Заполнить тексты `event.description` (lead)
  По 200+ событиям описания сейчас 1–2 предложения. Расширить хотя бы у популярных людей. Это про lead, не про `sections` (sections заполнены в сессии 2026-05-04).

- [ ] **R3** Рабочие фото для всех 43 человек
  Текущие URL'ы в `public/data/people.json` часто 404. Заменить на стабильные ссылки с Wikimedia Commons / Wikipedia REST API thumbnail (Public Domain / CC). Сделать одноразовый скрипт обновления.

- [ ] **R4** Особый попап для связанных событий (попап-пара)
  Когда событие связано с другим человеком через `connections[]` обоих, при клике открывать особый попап про их совместное событие, не два отдельных. Дизайн пока не нарисован — до готовности показываем стандартный попап.

- [ ] **R6** Sticky captions при vertical scroll
  Названия мировых событий и метка века должны прилипать к верху Timeflow при vertical scroll. Требует структурного рефакторинга — вынос captions из event-блоков в отдельный `<div class="timeflow__captions-bar">` с `position: sticky; top: 0`. Простой CSS-подход (sticky внутри absolute) не работает чисто.

- [ ] **R7** Категория `musician`
  Токен `--surface-person-musician` (✅ есть как `var(--toxic-100)`); variant `Event caption / Category=Musician`; добавить музыкантов в `people.json`, проверить `roster.json`.

- [ ] **R8** Полировка анимаций по всему сайту
  Появление event-caption и connection при hover, slide-in/out sidebar, scroll-by-zoom переходов, любых state-переходов.

- [ ] **R9** Empty state — для главного экрана (фильтр пустой) и для сайдбара.

- [ ] **R10** Loading / fetch state — пока подгружаются данные.

- [ ] **R12** Walt Disney = businessman — сверить
  В попапе он голубой (`#3ed6f5`). Проверить в `public/data/people.json` / `roster.json`.

- [~] **R13** «Выбрать все» в сайдбаре
  Опция, включающая все чекбоксы (вероятно глобально по всему сайдбару, не на уровне отдельной группы — уточнить при реализации). Сейчас чекбокс «Все» работает только на уровне раскрытой группы.

- [ ] **R14** Кастомный скролл везде
  В Timeflow кастомный стиль скролла уже есть. В сайдбаре и других местах всё ещё виден дефолтный браузерный — на некоторых компьютерах особенно заметно. Привести все скроллы к единому стилю.

- [ ] **R15** Аудит связей и событий
  Два сценария:
  - **Реальная связь есть в истории, у нас её нет.** Пример: Троцкий и Фрида Кало встречались, но связи в `people.json` не зафиксировано.
  - **У нас есть событие про другого человека без двусторонней связи.** Пример: у Густава Малера есть событие «Встреча с Фрейдом», но в `connections[]` обоих нет ссылки друг на друга.
  Нужен общий проход — в первую очередь по событиям с упоминанием других людей.

- [ ] **R16** Дополнить скиллы `/add-person`, `/add-event`, `/check`
  Цель — повышение достоверности и консистентности базы: правильная структура текста на каждом событии, корректные двусторонние связи между людьми, проверка соответствия событий и связей. Сейчас скиллы делают минимум.

- [ ] **R17** Hover-механика связанных событий
  Hover на любую часть связи (линия / точка) → подсветить всю связь: линия + обе точки + имена обоих участников. Имена показываются даже когда соответствующая точка за пределами viewport по вертикали. Симметричное поведение — триггер любой части даёт одинаковый эффект.

- [ ] **R19** Интерактивный таймлайн в попапе события
  В попапе события есть мини-таймлайн жизни персоны с маркером текущего события. Сделать интерактивным: клик на другую точку на этом таймлайне → попап переключается на соответствующее событие того же человека, без закрытия и переоткрытия.

- [ ] **R20** Оптимизация фотографий
  Уменьшить вес фото для быстрой загрузки попапов. Возможные направления: сжатие / `webp`, размер по фактическому use-case, lazy-loading.

- [ ] **R21** Возраст при hover — рядом с названием, не поверх
  При наведении на event на Timeflow цифра «X лет» сейчас отображается поверх caption события. Нужно отображать рядом (сбоку), чтобы не перекрывало текст.

- [ ] **R22** Новая версия дизайна попапа события.

- [ ] **R23** Drag-функционал на ползунке zoom
  Сейчас zoom меняется только кликом на +/−. Добавить возможность тащить ползунок мышью (drag).

- [~] **R24** Поменять ховер на компоненте selector
  Текущий ховер на `.selector` (компонент в сайдбаре, см. `styles/components/selector.css`) пользователю не нравится. Конкретный новый дизайн уточнить при реализации.

- [~] **R25** Анимация при раскрытии списка с чекбоксами в сайдбаре
  Сейчас при клике на selector список чекбоксов (`.sidebar__list`) появляется мгновенно. Нужна плавная анимация раскрытия (fade / slide-down / collapse — тип уточнить при реализации).

- [ ] **R28** Пересмотреть текстовые токены (типографика + цвет текста)
  Общий ревью всех text-related токенов:
  - **Типографика:** `--font-h1-*`, `--font-h2-*`, `--font-lead-*`, `--font-body-*`, `--font-body-s-*`, `--font-semibold-14-*` — family / weight / size / line-height.
  - **Цвет текста:** `--text-primary`, `--text-primary-hovered`, `--text-primary-active`, `--text-secondary`, `--text-tertiary`, `--text-century`, `--text-white`, `--text-black`.
  Пройтись по обеим группам, проверить актуальность ролей и значений; при необходимости — добавить/удалить/переименовать.
