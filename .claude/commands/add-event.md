# /add-event

Добавляет мировое событие в `public/data/events.json`.

## Аргумент
Название события передаётся после команды: `/add-event Первая мировая война`

## Алгоритм

1. **Прочитай** `public/data/roster.json`, `public/data/events.json` и `public/data/_schema.js`.
   - Если у события уже `"added": true` в ростере — сообщи, что оно уже есть в базе, и остановись.
   - Если события **нет в `roster.json`** — это нормально, просто добавь его в ростер вместе с записью в базу.

2. **Исследуй** событие через веб-поиск:
   - Точное название на русском
   - Даты: одиночный год (`year`) или диапазон (`startYear` + `endYear`)
   - Описание для попапа: 2–5 предложений на русском — что произошло, почему важно, чем закончилось

3. **Сформируй запись** строго по схеме из `public/data/_schema.js`:
   - Для длящихся событий (война, эпоха, кризис) — используй `startYear` + `endYear`, поле `year` = null
   - Для точечных событий (открытие, убийство, революция одного года) — используй `year`, поля `startYear` и `endYear` не включай
   - `id` — snake_case, краткий, уникальный (например: `wwi`, `russian_revolution`, `gold_rush`)

4. **Покажи результат** пользователю в виде JSON-блока перед добавлением. Кратко прокомментируй категорию и временной охват.

5. **Спроси подтверждение** перед записью в файл: "Добавить в events.json?"

6. **После подтверждения** — добавь запись в `public/data/events.json` и установи `"added": true` для этого события в `public/data/roster.json`.

## Важно
- Только события в диапазоне `START_YEAR`–`END_YEAR` (см. `public/data/_schema.js`) или пересекающиеся с ним.
- Если событие началось до `START_YEAR` или закончилось после `END_YEAR` (см. `public/data/_schema.js`) — обрезаем по границам диапазона в описании, но в полях `startYear`/`endYear` пишем реальные годы.
- Только факты, подтверждённые источниками.
