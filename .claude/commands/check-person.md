# /check-person

Полный фактчек всех событий одного человека + поиск связей по всему его профилю.

См. общий контракт: `.claude/commands/_research-protocol.md`. Это batch поверх `/check-event`.

## Аргумент

`/check-person <person-id>` — например, `/check-person einstein`.

## Алгоритм

1. **Получи профиль человека:**
   ```bash
   jq '.[] | select(.id == "<person-id>")' public/data/people.json
   ```
   Извлеки: `name`, `fullName`, `born`, `died`, `bio`, `events[].year`, `connections[]`.

2. **Сверь основные факты** (даты жизни, национальность, краткая биография) с Wikipedia ru. Один WebFetch на всего человека.

3. **Прогон по событиям:**
   - Для каждого `event` в `events[]` — выполни логику `/check-event` (см. этот файл, пункты 3–6).
   - Накопи находки в локальную переменную/буфер.

4. **Сводный отчёт.** Формат:

   ```
   # check-person: <name> (<count> событий)

   ## Профиль
   ✅/⚠️/❌ born/died/bio — <вердикт>

   ## События

   ### <year>: <title>
   <короткий вердикт по lead + sections>
   [подробности — в .scratch/check-<date>-<person-id>.md если событий 5+]

   ### <year>: <title>
   ...

   ## Связи (R15)
   Найдены новые упоминания:
   - <person-id-A> — в источнике <X> по событию <year>
   - <person-id-B> — ...

   ## Итог
   ✅ N подтверждено, ⚠️ M уточнить, ❌ K противоречий, 🔗 P новых связей.
   ```

5. **Если событий 5+** — детали по каждому событию пиши в `.scratch/check-<YYYY-MM-DD>-<person-id>.md`, в чат — только сводку. Создай папку `.scratch/` если её нет.

6. **Применение правок** — пользователь выбирает (`all` / `none` / список номеров). См. `_research-protocol.md`.

7. **После всех правок** — `/check`.

## Бюджет

Один прогон `/check-person` ≈ 4–6 WebFetch (общий профиль + по 1 на событие). Не превышай — лимиты дорого.

Если у человека 10+ событий — раздели на 2 запуска: «`/check-person <id>` первая половина», «вторая половина».
