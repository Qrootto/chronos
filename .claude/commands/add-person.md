# /add-person

Добавляет исторического персонажа в `public/data/people.json`.

## Аргумент
Имя человека передаётся после команды: `/add-person Зигмунд Фрейд`

## Алгоритм

1. **Прочитай** `public/data/roster.json`, `public/data/people.json` и `public/data/_schema.js`.
   - Если у человека уже `"added": true` в ростере — сообщи, что он уже есть в базе, и остановись.
   - Если человека **нет в `roster.json`** — это нормально, просто добавь его в ростер вместе с записью в базу.

2. **Исследуй** персонажа через веб-поиск:
   - Полное имя на языке оригинала
   - Даты жизни
   - Национальность
   - Категория: один из значений `PERSON_CATEGORIES` (см. `public/data/_schema.js`)
   - Краткая биография 2–3 предложения (на русском)
   - Все значимые события жизни **строго в диапазоне `START_YEAR`–`END_YEAR`** (см. `public/data/_schema.js`): год, короткое название для таймлайна (до 4–5 слов), описание для попапа (2–5 предложений на русском)
   - Ссылка на портрет в общественном достоянии (Wikimedia Commons предпочтительно)

3. **Найди связи** с людьми, которые уже есть в `people.json`:
   - Просмотри всех текущих персонажей
   - Для каждой реальной связи: год, тип — один из значений `CONNECTION_TYPES` (см. `public/data/_schema.js`), описание
   - Добавь связь **у обоих** людей (у нового и у существующего в файле)

4. **Сформируй запись** строго по схеме из `public/data/_schema.js`.

5. **Покажи результат** пользователю в виде JSON-блока перед добавлением. Кратко прокомментируй: сколько событий нашёл, какие связи обнаружил.

6. **Спроси подтверждение** перед записью в файл: "Добавить в people.json?"

7. **После подтверждения** — добавь запись в `public/data/people.json`, обнови связи у уже существующих персонажей если они появились, и установи `"added": true` для этого человека в `public/data/roster.json`.

## Важно
- Только факты, подтверждённые источниками. Если год события неточен — указывай "около" в описании, но в поле `year` ставь ближайший достоверный год.
- Не выдумывать связи. Только реально задокументированные знакомства, встречи, переписка, совместные работы.
- Если человек родился до `START_YEAR` или умер после `END_YEAR` (см. `public/data/_schema.js`) — это нормально, просто показываем только события в этом диапазоне.
