# /check-event

Фактчек одного события человека через внешние источники + поиск новых связей.

См. общий контракт: `.claude/commands/_research-protocol.md`.

## Аргумент

`/check-event <person-id> <year>` — например, `/check-event lenin 1905`.

Если событий с этим годом у человека несколько (диапазон + точечное) — спроси какое именно.

## Алгоритм

1. **Достань событие точечно** (не читай весь `people.json`):
   ```bash
   jq '.[] | select(.id == "<person-id>") | {name, fullName, events: [.events[] | select(.year == <year>)]}' public/data/people.json
   ```

2. **Достань текущие connections этого человека:**
   ```bash
   jq '.[] | select(.id == "<person-id>") | .connections' public/data/people.json
   ```

3. **Опроси Wikipedia (ru → en)** по событию. Контекст запроса = `<fullName> <year>` или `<event.title> <fullName>`. Если ru дала достаточно — на en не дёргайся.

4. **Сверь** наш `description` и каждую секцию `sections[]` с источником:
   - Точные годы, цифры, имена — совпадают?
   - Формулировки не выдуманы?
   - Если в нашем тексте конкретика, которой нет в источнике — ⚠️ или ❌.

5. **Если Wikipedia коротко/спорно** — дёрни Britannica или SEP (по типу персонажа). **Один доп. источник, не два.**

6. **Поиск связей** (см. `_research-protocol.md` → «Поиск связей»):
   - Загрузи roster: `jq '.people[] | .id' public/data/roster.json`
   - Также имена: `jq '.[] | {id, name, fullName}' public/data/people.json`
   - В текстах источников ищи упоминания этих имён.
   - Сверь с текущим `connections[]`. Пропуски → в отчёт.

7. **Отчёт по формату** из `_research-protocol.md` → секция «Формат отчёта».

8. **Применение правок** — по `_research-protocol.md` → секция «Применение правок». Без молчаливых правок.

9. **После применения** — `/check` (целостность данных).

## Что НЕ делает

- Не дёргает все 4 источника на каждый факт.
- Не пишет в `public/data/*.json` без подтверждения пользователя.
- Не оценивает «художественность» текста — только проверяемые факты.
