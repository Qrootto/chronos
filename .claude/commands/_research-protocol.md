# Research Protocol (общий контракт скиллов /check-*)

Этот документ — **не скилл**. Это общий контракт, на который ссылаются `/check-event`, `/check-person`, `/audit-connections`. Подчинение проектному правилу «Достоверность» в `CLAUDE.md`.

---

## Источники

Опрашивай по убыванию приоритета. **Не дёргай все четыре** на каждый факт — это разорит лимиты.

| # | Источник | Когда использовать | Доступ |
|---|---|---|---|
| 1 | Wikipedia (ru → en) | baseline всегда. Сначала ru, если коротко/нет — en | `WebFetch https://ru.wikipedia.org/wiki/<Имя>` |
| 2 | Britannica | если Wikipedia коротко, спорно, или у нас «черновик» противоречит | `WebFetch https://www.britannica.com/biography/<Name>` |
| 3 | Stanford Encyclopedia of Philosophy | для философов/учёных, особенно по идеям и трудам | `WebFetch https://plato.stanford.edu/entries/<slug>/` |
| 4 | Internet Archive | для первоисточников, мемуаров, писем, исторических газет | `WebSearch site:archive.org <запрос>` |

**Стратегия по умолчанию:** Wikipedia ru+en → если хватает, останавливаемся. Britannica/SEP/Archive — только когда есть пробелы или сомнения.

**Wikimedia Commons API** — отдельный канал для фото и метаданных (см. `/fix-photos`).

## Формат отчёта (в чат, не в файл)

Для **одного события**:

```
## <person.name>, <event.year>: <event.title>

**Lead (description):**
✅/⚠️/❌ <вердикт> — <короткое пояснение>
[если ⚠️/❌] Предложение: <как переписать>

**Sections:**
- ✅/⚠️/❌ <section.title> — <вердикт>
  [если ⚠️/❌] Предложение: <как переписать>

**Найдены связи (R15):**
- <person-id из roster> — <в каком источнике упомянут, в каком контексте>
- [или] Не найдено новых.

**Источники:** Wikipedia ru, Britannica
```

Легенда:
- ✅ подтверждено источниками
- ⚠️ неточность (год, цифра, формулировка)
- ❌ противоречие или не нашёл подтверждения

**Если batch на 5+ событий** — пиши в `.scratch/check-<date>-<person>.md`, в чат — только сводка.

## Поиск связей

Это **киллер-фича проекта**. Каждый прогон должен это делать.

1. Загрузить `public/data/roster.json` → собрать список всех имён людей (с алиасами, если есть `fullName`).
2. В тексте каждого источника (Wikipedia/Britannica/...) для текущего события — **искать упоминания** этих имён.
3. Для каждого найденного:
   - Если уже есть в `connections[]` у текущего человека → пропустить.
   - Если нет → **предложить добавить** в отчёт.
4. Связь — двусторонняя. При добавлении правится у обоих (как в `/add-person` шаг 3).

Тип связи (`CONNECTION_TYPES` из `_schema.js`) — определяется по контексту источника. Если непонятно — спросить пользователя.

## Применение правок

**Скиллы НЕ пишут в `public/data/*.json` автоматически.** После отчёта:

1. Покажи список предлагаемых правок (нумерованный).
2. Спроси пользователя: «Применить какие? (`all` / `none` / `1,3,5`)».
3. Только после ответа — Edit на конкретные поля.
4. После Edit запусти `/check` для проверки целостности.

## Экономия токенов

- WebFetch — дорогой. Один источник на факт по умолчанию (Wikipedia). Дополнительные — только при сомнении.
- Не читай весь `people.json` (он большой). Используй `jq` или `grep` для точечного доступа: `jq '.[] | select(.id == "<id>")' public/data/people.json`.
- Batch-скиллы (`/check-person`) — пиши отчёт в файл, в чат — только итог.
