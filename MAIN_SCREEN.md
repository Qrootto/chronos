# Часть 1. Глобальная модель

## Главная идея проекта

Past Simple — интерактивный временной таймлайн исторических личностей. Главная цель: показать, что разные люди жили в одну эпоху и были связаны. Пользователь видит линии жизни выбранных людей на одной шкале лет, наводит на события, видит связи между людьми, открывает попап с описанием.

## Координатная система

**Временная ось — единственная «правда» экрана.** Все позиции (имена, линии жизни, точки событий, мировые события, метки веков, метки шкалы) считаются из одной формулы:

```
x = (year − START_YEAR) × pxPerYear
```

- `START_YEAR` — самый ранний показанный год основной шкалы. Сейчас **1850**.
- `END_YEAR` — самый поздний. Сейчас **1950**.
- `pxPerYear` — масштаб; меняется зумом.
- `Timeflow.width = (END_YEAR − START_YEAR) × pxPerYear`

Координаты Timeflow и Year scale **обязаны совпадать пиксель-в-пиксель** для одинаковых лет — это главный инвариант.

## Layout главного экрана

```
┌─────────────────────────────────────────────────┐
│  Header (всегда сверху, фикс высоты)             │
├─────────────────────────────────────────────────┤
│ S │  Timeflow                                    │
│ i │  ─ метки веков сверху (всегда видны)         │
│ d │  ─ имена слева/у начала линии                │
│ e │  ─ линии жизни (life-line)                   │
│ b │  ─ точки событий (event-dot)                 │
│ a │  ─ вертикальные мировые события (event)      │
│ r │                                              │
│   │  Year scale (внизу, всегда видна)            │
│   │                                  Zoom (+/−)  │
└─────────────────────────────────────────────────┘
```

- **Sidebar** скрыт по умолчанию, выезжает по burger.
- **Timeflow + Year scale** — один композит с общей системой координат и общим горизонтальным скроллом (одна обёртка `.canvas__scroll → .canvas__inner`).
- **Popup** открывается поверх всего при клике на event-dot.
- Layout **полностью flexible**: высота/ширина считаются от размера окна, нет хардкода размеров.

## Header

### Суть
Постоянная шапка сверху. Содержит вход в фильтр (burger), компас по большой временной картине (timeline-mini в центре), вход в инфо (info справа).

### Поведение
- Высота фикс 64px, ширина = 100% окна.
- Burger клик → toggle Sidebar.
- Info клик → попап «О проекте» (дизайн-долг).

## Sidebar

### Суть
Фильтр выбора людей и событий. Через чекбоксы пользователь добавляет/убирает линии жизни на Timeflow.

### Поведение
- **На всю высоту окна**, **поверх Header** (overlay, position: fixed, z-index выше Header). Header не сдвигается, остаётся на месте — sidebar его перекрывает.
- При открытии **толкает Timeflow вправо** на свою ширину (288px) — canvas получает `margin-left: 288px`. Header при этом остаётся на месте, потому что он вне `.main-canvas`.
- Скрыт по умолчанию (`transform: translateX(-100%)`). Выезжает слева направо по клику на burger.
- Закрытие — клик на close-крестик внутри Sidebar. Hover у close такой же, как у burger/info: `icon-btn--ghost`.
- **7 групп**: «События» + 6 категорий людей (Художники, Писатели, Музыканты, Учёные, Бизнесмены, Политики).
- Каждая группа — раскрываемый селектор. В Default состоянии у chevron — стрелка вниз (`▼`); в Active (раскрытом) — стрелка вверх (`▲`).
- **Счётчик в заголовке группы**: справа от названия (gap 4px, цвет `--text-primary`, тот же шрифт, что у названия) пишется либо **число** выбранных в группе (если выбраны не все), либо «**все**» (если все). Один и тот же формат для всех групп — включая «События».
- В раскрытой группе — список с чекбоксами. Чекбокс включает/выключает соответствующий объект на Timeflow (для группы «События» — мировое событие, для категорий — линию жизни персоны).

### Поведение раскрытия
- **Только одна группа открыта в момент времени.** При раскрытии новой группы предыдущая открытая закрывается автоматически.
- **Внутренний скролл в раскрытой группе.** Если открытая группа не помещается на экран по вертикали (вместе с заголовками всех остальных групп), то скролл появляется **внутри списка чекбоксов** раскрытой группы, а не у всего sidebar. Заголовки всех групп (закрытых) всегда видны — открытая группа сжимается, остальные группы остаются на месте.

### Стартовое состояние
- Показываем **рандомный набор** людей (выбираем при загрузке).
- Все мировые события включены по умолчанию («События: все»).

### Чекбокс «Все» в группе
В каждой раскрытой группе **первый элемент списка — чекбокс «Все»**. Управляет всей группой:
- **Включён** (`checkbox--checked`) — все элементы группы выбраны.
- **Выключен** (default) — ни один не выбран.
- **Deselect** (`checkbox--deselect`, новое состояние) — часть выбрана, часть нет. Выглядит как заполненный квадрат brand-цвета, внутри — горизонтальная линия (вместо галочки), `2px` высотой со скруглёнными краями, отступы по `3px` от краёв чекбокса.
- Клик по «Все»: если он не unchecked → снимает выделение со всех; иначе → выбирает все.

## Timeline-mini

### Суть
Уменьшенная карта-ориентир в шапке. Помогает пользователю понять, **где он сейчас находится относительно большой картины времени** (от 18 до 21 века). Без неё пользователь, погружённый в основной timeflow на интервале 1850–1950, не понимает, как этот интервал соотносится с эпохами в целом.

### Координатная система
Своя, независимая от основной формулы Timeflow:

```
x_mini = (year − 1700) × 1px/year
```

- `START_OF_MINI = 1700`, `END_OF_MINI = 2100` (4 века).
- `1 год = 1 px`. `1 век = 100 px`.
- Общая ширина компонента = 400px.

### Поведение
- **Дивайдеры** между веками: на x=0, 100, 200, 300, 400 (границы XVIII/XIX/XX/XXI).
- **Метки веков** (XVIII, XIX, XX, XXI): центрируются между дивайдерами.
- **Time** — полоса фактического времени (от 1700 до сегодняшнего года). Имеет красную правую границу `border-lineend` (метка «сейчас»).
- **Focus** — прозрачная рамка, **отражение видимой области основного Timeflow**. Динамическая:
  - `focus.width = visibleYearsInTimeflow × 1px = ширина_окна_timeflow_в_годах`.
  - `focus.left = (currentViewportStartYear − 1700) × 1px`.
  - При **resize окна** ширина focus меняется (видно меньше/больше лет → focus уже/шире).
  - При **горизонтальном скролле** Timeflow → focus двигается по timeline-mini синхронно.
- **Fade слева** — прогрессивный blur для области далёкого прошлого (визуальная подсказка «туда мы пока не уходим»). Fade блюрит только полосу `time` под собой, **дивайдеры и метки веков остаются резкими** (включая самый левый дивайдер на x=0).

### Связь с Timeflow
Timeline-mini **созависим** с основным Timeflow:
- viewport основного Timeflow → положение и ширина focus.
- (В будущем) перетаскивание focus за рамку → скролл основного Timeflow в эту позицию.

## Timeflow

### Суть
Центральная сцена. Здесь живут линии жизни выбранных людей и мировые события. Это «полотно времени», на котором всё привязано к единой шкале.

### Поведение
- Ширина = `(END_YEAR − START_YEAR) × pxPerYear`.
- Высота = всё доступное место под Header и над Year scale (адаптивна, считается от высоты окна).
- Контент **обрезается границами контейнера** (overflow: hidden). Если линия не помещается по вертикали — она обрезается границей Timeflow, не нижней границей окна.
- Внутри:
  - Метки веков сверху (всегда видны, top: 8px от верха Timeflow).
  - Линии жизни (life-line) — горизонтальные.
  - Точки событий (event-dot) — на линиях.
  - Мировые события (event) — вертикальные блоки от верха (top: 23px = низ метки века + 4px gap) до низа Timeflow.

---

### Имена на линиях жизни (sticky)

#### Суть
Имя должно быть видно пользователю всегда, пока он в принципе **может видеть линию жизни этого человека** (хоть кусочек). Если же линии вообще не видно — имя превращается в указатель «там, в той стороне, есть человек».

#### Поведение
Три случая в зависимости от того, видна ли линия в текущем viewport:

**1. `born` виден в viewport** (не уехал за край):
- Имя стоит на координате `born`, выше линии. Без стрелки.

**2. Линия пересекает viewport, но `born` за его краем:**
- Имя приклеено к **видимому концу линии** (к левому краю viewport — если уехал left, к правому — если уехал right).
- **Стрелки нет** — пользователь видит саму линию, по ней понятно куда она тянется.

**3. Линия полностью за viewport (не пересекает его):**
- Имя приклеено к ближайшему краю viewport.
- Рядом — треугольная стрелка в сторону линии: `◀ Имя` (линия слева) или `Имя ▶` (линия справа).
- Смысл: пользователь линии не видит, стрелка говорит «там есть человек».
- Имя со стрелкой имеет **opacity 0.32** — приглушённый «фоновый указатель», не отвлекает.
- Стрелка нарисована SVG-полигоном с **сглаженными углами ~2px** — не острый пик.

**Когда вся жизнь оказалась за пределами viewport в направлении, в котором пользователь смотрит** (например, человек умер до того года, что виден в viewport) — имя пропадает совсем.

---

### Вертикальное распределение людей

#### Суть
Линии жизни должны всегда **визуально занимать всю высоту Timeflow равномерно**. Никаких пустот сверху или снизу. При добавлении человека линии сжимаются (как гармошка), при убирании — растягиваются.

#### Поведение
Для N людей и доступной высоты H, y-позиция линии i (от 0 до N−1):

```
y(i) = H × (i + 1) / (N + 1)
```

Это даёт:
- 1 человек → ровно посередине (H/2).
- 2 → 1/3 и 2/3.
- 3 → 1/4, 2/4, 3/4.

**Минимальный шаг = 43px.** Если `H / (N + 1) < 43px` — фиксируем шаг = 43px и появляется вертикальный скролл (контент становится выше viewport).

### Очерёдность по категориям (сверху вниз)
1. Художники (artist)
2. Писатели (writer)
3. Музыканты (musician)
4. Учёные (scientist)
5. Бизнесмены (businessman)
6. Политики (politician)

Внутри одной категории — порядок добавления (стабильный).

**При resize окна** — шаг пересчитывается, линии смещаются.

### Hover на линию жизни

- Линия становится толще: 3px → 5px и подсвечивается.
- Растёт **симметрично от центра**: на 1px вверх и 1px вниз. Центр линии остаётся на месте.

## Year scale

### Суть
Нижняя шкала годов, привязанная к той же координатной системе, что Timeflow. Подпись каждого года — чтобы пользователь сразу видел, на какой год он смотрит.

### Поведение
- Метка ставится на **каждый** год шкалы. Каждый год — это отдельный компонент `.year` (`__dot` + `__label`, оба центрированы в боксе 20×17).
- Сама шкала прижата к краям Timeflow: 1850 на самом левом краю, 1950 на самом правом. Крайние метки **частично обрезаются** левым/правым краем экрана — это допустимо. Не сдвигаем их специально, чтобы не наезжали на соседние.

### Hover на год
При движении курсора **по любому месту Timeflow** (не только над самой меткой) год прямо под курсором переходит в состояние `--hovered` (точка ярче, label ярче). Это нужно, чтобы пользователь всегда видел, на какой год сейчас смотрит.

**Hover-area для года Y по горизонтали** — от центра-между-(Y−1, Y) до центра-между-(Y, Y+1). То есть ровно полпролёта `pxPerYear` слева и справа от центра года. По вертикали — вся высота Timeflow.

Реализуется JS-обработчиком `mousemove` на области Timeflow: `Math.round((scrollLeft + mouseX) / pxPerYear) + startYear` даёт год под курсором (округление автоматически даёт нужные границы).

## Event (мировые события)

### Суть
Вертикальные блоки на временной шкале, обозначающие исторические события (войны, кризисы, открытия). Помогают увидеть контекст эпохи у выбранных людей.

### Положение
- **Сверху:** `top: 23px` (= низ метки века 19 + 4px gap).
- **Снизу:** `bottom: 4px` от верха Year scale (визуальный gap до точек годов).
- **По горизонтали:** идеально центрировано на координате года. Tick года (точка в Year scale) находится центром на координате года; левый/правый край event-блока (1px-линия) центром тоже на координате года. То есть `left = yearToX(year) − 0.5` для 1px-линии. Для widths-блоков (one-year, several-years): левый край на `yearToX(startYear) − 0.5`, правый — на `yearToX(endYear) + 0.5`.

### Тип по длительности
- `startYear == endYear` → `event--less-than-year` (1px).
- `endYear − startYear ≤ 1` → `event--one-year`.
- `endYear − startYear > 1` → `event--several-years`.

### Подпись над event-блоком (event__caption)

#### Суть
Пользователь должен по hover увидеть название события. Метки веков (XIX, XX) видны всегда, поскольку это «точки ориентации» в шкале.

#### Положение и поведение
- На **4px** над верхом event-блока, **центрировано по горизонтали** относительно event-блока (для линии 1px центр совпадает с координатой года; для блока several-years — середина между его левым и правым краем).
- Стиль: Inter SemiBold 14, `text-primary-hovered`.
- **Видимость:**
  - Default — скрыта.
  - На `:hover` события — появляется.
  - Для `event--secondary` (дивайдер между веками) с модификатором `event__caption--persistent` — всегда видна (метка века).
- **Suppression**: если появляющаяся hover-caption визуально пересекается с persistent caption (метка века), persistent временно скрывается (класс `.is-suppressed`). При уходе курсора — возвращается.

## Hover на event-dot

### Суть
Пользователь должен навестись на любое событие в жизни любого человека и **сразу увидеть, что это за событие**. Без клика, без ожидания — просто навёл, прочитал.

### Поведение
1. Точка переходит в `--hovered`: размер увеличивается с 9px до 15px (радиус +2). Внутри — точка цвета `surface-bg` (= фон Timeflow), радиус +2 относительно прежней «дырки». Визуально это «бублик с маленькой точкой того же цвета, что фон».
2. Рядом появляется `event-caption` с названием события.

### Положение event-caption

У плашки event-caption один из углов имеет **наименьшее скругление** (≈1px) — это её «стрелка». Стрелка **всегда смотрит на точку**, к которой относится caption.

Соответственно:
- `event-caption--right` — caption справа от точки, стрелка снизу-слева (`bl ≈ 1`).
- `event-caption--left` — caption слева от точки, стрелка снизу-справа (`br ≈ 1`).

**По умолчанию — `--right`** (caption справа-сверху от точки).
**Переключаем на `--left`** (слева-сверху) если caption справа:
- уходит за правый край viewport, ИЛИ
- перекрывает что-то важное (имена других людей, названия мировых событий).

То есть всегда стараемся показать caption там, где он не мешает.

**Точная привязка к hovered-точке:**
- Нижняя граница caption — на одном уровне с **верхней границей hovered-dot** (15×15).
- Левая граница caption (`--right`) — на одном уровне с **правой границей hovered-dot**.
- Зеркально для `--left` (правая граница caption ≡ левая граница hovered-dot).

## Connections (линии связи)

### Суть
Связь между людьми — главная фишка проекта (показать, что разные люди были связаны через события). Связь **видна всегда** (когда оба человека и оба event-dot отрисованы) — это важно показывать пользователю с первого взгляда, какие люди вообще связаны. При hover события — связь подсвечивается gradient'ом из категорий двух людей. Симметрично: hover на любую из двух точек активирует ту же связь.

### Состояния
- **Default** — 1px дугообразная линия цвета `--connection-default` (= `--light-24`). Нарисована всегда, пока оба связанных человека и оба их события на одном году в DOM.
- **Hovered** — линия красится в gradient: цвет верхнего человека → цвет нижнего (по категориям).

### Форма линии
Связь не идеально вертикальная, а **слегка дугообразная** — quadratic Bezier с глубиной дуги 15px (control point на 30px правее прямой между точками). Это нужно, чтобы связи не сливались с границами мировых событий и другими вертикальными линиями на Timeflow.

### Точки выхода/входа
Connection выходит **не из центра точки, а из её угла** по диагонали «левый-нижний → правый-верхний» (диагональ делит точку пополам):
- У верхней точки (которая выше на экране) — линия выходит из её **нижне-правого** угла.
- У нижней точки — линия входит в её **верхне-правый** угол.

Точка выхода/входа = центр event-dot ± `R/√2` (где R — радиус видимой части default-точки = 4.5px, т.е. offset ≈ 3.18px). Это делает connection визуально отделённой от самих точек.

### Z-порядок
Connections **над** life-line, но **под** event-dot — связь видна над линиями жизни (не смешивается с ними), но точки рисуются поверх связи.

### Условие появления связи (Вариант 3)
Связь рисуется **только если у обоих людей в данных есть событие в один и тот же год**. Это обеспечивается данными: при добавлении связной пары событие создаётся у обоих (например, «А женился на Б» в `events[]` обоих, и `connections[]` обоих ссылаются друг на друга с тем же годом).

### Поведение
- При hover на event-dot у personA в год Y:
  - Среди `personA.connections` находим элементы с `year === Y`.
  - Для каждого, если связанный personB **сейчас отображён** в Timeflow и у него есть свой event-dot на том же year:
    - Рисуется `.connection` — вертикальная 1px-линия от центра точки А до центра точки Б.
    - Цвет — gradient от категории-А к категории-Б.
    - Анимируется плавно при появлении.
    - На точке Б тоже появляется hover-state и event-caption.
- На обоих концах связи видны event-caption (по правилам выше).
- При уходе курсора — связь и captions плавно исчезают.

### Sticky-Б при вертикальном overflow
Если по вертикали Timeflow появился скролл и связанный человек Б сейчас **за пределами viewport** (выше или ниже видимой области), то:
- В момент hover персона Б временно **«приклеивается» к ближайшему краю Timeflow** (верхнему или нижнему — в зависимости от того, где она находится).
- **Точка ghost** располагается на расстоянии **2px** от верхнего/нижнего края Timeflow (не вплотную).
- **Имя ghost** — слева от точки, на расстоянии **4px** (вертикально центрировано с точкой).
- Connection рисуется к этой ghost-точке.
- При уходе курсора ghost-элементы убираются.
- Логика: связь — главная фича, она должна быть видна всегда, даже если по геометрии экрана прямого пути нет.

### Несколько sticky-имён на одной стороне
Если у события несколько связей и **больше одного** связанного человека за тем же краем viewport (например, два за верхним краем), их ghost-точки и имена **не накладываются друг на друга** — следующая ghost-пара сдвигается **на 16px вниз** (для верхнего края) или **на 16px вверх** (для нижнего). Шаг 16 = высота имени 14 + 2px gap, чтобы и dot'ы, и имена шли с зазором ≥ 2px.

## Скролл

### Суть
Один общий скроллер по горизонтали для Timeflow и Year scale (они привязаны к одной шкале). Вертикальный — отдельный, у самой Timeflow, если людей слишком много.

### Поведение
- **Горизонтальный** (по времени): общая обёртка `.canvas__scroll → .canvas__inner` имеет `overflow-x: auto`. Timeflow и Year scale внутри двигаются синхронно.
- **Вертикальный** (по людям): когда `H / (N + 1) < 43px` — у `.canvas__timeflow-area` `overflow-y: auto`. **Header и Year scale остаются на месте** — скроллится только содержимое Timeflow.

### Нижняя граница Timeflow
Нижняя граница Timeflow проходит ровно по нижнему краю мировых событий — не доходит до точек Year scale. Между low границей timeflow и tick'ами year есть визуальный gap.

## Zoom

### Суть
Управление масштабом времени. Растягивает или сжимает шкалу — как таймлайн в видеоредакторе.

### Положение на экране
- Над Year scale на **8px**.
- От правого края экрана **8px**.

### Шаги
**5 шагов** на треке. Текущее состояние сайта (`pxPerYear=40`) — **второй слева** шаг. Минимум (самый левый) — такой, чтобы между числами годов оставалось ~3px (с учётом ширины подписи ~30px → `pxPerYear ≈ 33`). Дельта между соседними шагами — постоянная.

Шаги (в `pxPerYear`): **33, 40, 47, 54, 61**.

### Поведение
- `+`/`−` или клик по треку перемещают thumb на следующий/предыдущий шаг (не плавно, а «прыжком» по шагам).
- Альтернатива: тянуть focus-рамку в timeline-mini.
- **Анимация thumb по шагам — быстрая, но плавная** (transition ~0.15s).
- **Сохранение центра viewport.** При смене масштаба год, который был под центром экрана, остаётся под центром экрана. Формула: `viewportCenterYear = startYear + (scrollLeft + viewportWidth/2) / oldPxPerYear; newScrollLeft = (viewportCenterYear − startYear) × newPxPerYear − viewportWidth/2`.

## Popup

(Старая геометрия в `chronos_popup_geometry.md` устарела — переписана 2026-05-04.)

### Суть
Карточка с подробным рассказом о событии. Открывается, когда пользователь клацает по event-dot на линии жизни персонажа. В попапе — заголовок «{Имя} в {год}», life-timeline персонажа, краткое описание (lead) и подробный текст с подзаголовками. В правой колонке — фото персонажа и блок «В это же время» (современники, у которых на этот год тоже есть события).

### Геометрия
- **Шторка справа налево.** `position: fixed`, прижата к правому, верхнему и нижнему краям экрана (`top:0, right:0, bottom:0`), без отступов.
- **Ширина — 85vw**. Слева остаётся 15vw видимого Timeflow.
- В этих 15vw слева **горизонтально центрируется выбранное событие** (год, на которое кликнули). Программно скроллим Timeflow так, чтобы center видимой части = центр 15vw = `7.5vw`.
- За попапом — `backdrop-filter: blur(4px)`.

### Анимация
- По клику на event-dot: `transform: translateX(100%) → 0`, плавно (~0.3s).
- По клику на close (или Esc): обратно в `translateX(100%)`.

### Затемнение
Под попапом — полноэкранный backdrop `.popup-backdrop` цвета `--surface-darkener` (= `--black-64`). Затемняет видимую часть main screen в 15vw слева. Click по нему = закрыть попап.

### Кнопка close
В правом верхнем углу попапа (20px от верха и от правого края) — `icon-btn--ghost` (тот же стиль что у burger/info/sidebar-close). Закрывает попап.

### Раскладка (двухколоночная)
- Контейнер: `display: grid`, `grid-template-columns: 2fr 48px 1fr`. `padding-left: 32px` (без `padding-right`/`padding-top`/`padding-bottom`).
- **Левая колонка** (2/3): отступ от верха попапа **64px**, контент скроллится по вертикали.
  - **Heading** (gap 32px между title и life-line):
    - Title: `Inter Regular 40`, white. Текст: «{Имя} в {год}».
    - Life-line: над линией три года (born / event_year / died), Inter Medium 14. Born — слева, died — справа, event_year — пропорционально на линии (по формуле `(year − born) / (died − born)`), цветом категории. Под линией на той же координате — event-dot цвета категории.
  - Lead: `Inter Regular 24`, line-height 34, opacity 80%. Краткое описание этого года.
  - Text: набор секций, gap 41px между секциями. Внутри секции gap 12px между заголовком и параграфом.
    - h2: `Inter SemiBold 24`, line-height 22.
    - p: `Inter Regular 16`, line-height 26, opacity 80%.
- **Правая колонка** (1/3): отступ от верха попапа **64px** (фото начинается оттуда же, что и левая колонка).
  - Photo: **всегда квадратное** (через `aspect-ratio: 1/1` + `background-image`). Если URL фото сломан — остаётся placeholder-фон, бокс остаётся квадратным.
  - Под ним — блок «В это же время» (gap 16 от фото).
  - Concurrent items: точка 5×5 + текстовый столбец (имя в цвете категории + описание мутным белым).

# Часть 3. Что это означает для кода

1. **Состояние** хранит `startYear`, `endYear`, `pxPerYear`, набор включённых людей, scrollLeft.
2. **Render-функции** для каждой части (Timeflow, Year scale, Timeline-mini, Sidebar) получают state + данные → расставляют DOM по правилам выше.
3. **Re-render** на:
   - изменение state (фильтр, zoom),
   - resize окна (для вертикального распределения и focus в timeline-mini),
   - scroll (для focus в timeline-mini и sticky-имён).
4. **Hover-обработчики** на event-dot — показывают caption, рендерят connection если есть связанный видимый человек.
5. **Sticky-имя** — на каждом scroll'е/resize'е пересчитывается, какая у имени позиция и какая стрелка.
